Zero-Shot Prompting

모델에게 예시를 전혀 제공하지 않고 지시만으로 태스크를 수행시키는 가장 기본적인 기법입니다.

Technique
PROMPT > 다음 영화 리뷰의 감정을 긍정/부정으로 분류하시오. REVIEW > "이 영화는 시간 낭비였다. 스토리가 너무 진부했다." OUTPUT > 부정 (Negative)
Level: Beginner

FAQ

자주 묻는 질문들

Q프롬프트 엔지니어링은 개발자만 배울 수 있나요?

코딩 지식이 없어도 배울 수 있는 커뮤니케이션 기술에 가깝습니다.

QGPT, Claude, Gemini 등 모델마다 프롬프트가 달라야 하나요?

핵심 원칙은 동일하지만 모델별 특성에 따라 미세 조정이 필요합니다.

QFew-Shot 프롬프트는 언제 사용해야 하나요?

정형화된 데이터나 특정 톤앤매너를 추출해야 할 때 예시를 제공하면 좋습니다.

QChain-of-Thought(CoT)는 항상 효과적인가요?

추론이 필요한 복잡한 문제에서만 효과적이며, 단순 작업에는 토큰 낭비입니다.

Q프롬프트가 길수록 결과가 좋아지나요?

길이보다 명확성과 밀도가 중요합니다. 불필요한 수식어는 혼란을 줍니다.

QSystem Prompt와 User Prompt의 차이는 무엇인가요?

System은 전체 행동 방식과 역할을 정의하고, User는 개별 대화에서의 구체적 요청입니다.

Common Pitfalls

자주 하는 실수들

카드 위에 마우스를 올려 실수의 해결책을 확인하세요. 작은 습관의 차이가 프롬프트 품질을 결정합니다.

Mistake 01

너무 모호한 지시

"좋은 글 써줘", "코드 고쳐줘"처럼 목적·대상·형식이 전혀 없는 지시는 엉뚱한 결과를 냅니다.

뒤집어서 해결책 보기
Solution

Context + Task 명시

"마케터를 위한 SNS 캡션을 인스타그램 스타일로, 150자 이내로 3가지 버전으로 작성해 줘."

Role + Goal + Format + Limit
Mistake 02

역할 부여 없음

AI에게 '누구처럼' 행동할지 알려주지 않으면 일반적이고 밋밋한 답변만 돌아옵니다.

뒤집어서 해결책 보기
Solution

Persona 설정

"당신은 10년 경력의 UX 라이터입니다. 전환율 최적화 관점에서 랜딩 페이지 카피를 작성해 주세요."

Act as [Expert]
Mistake 03

출력 형식 미지정

결과물의 형식(JSON, 표, 목록 등)을 지정하지 않으면 매번 다른 형태로 응답이 옵니다.

뒤집어서 해결책 보기
Solution

Format 명시

"결과는 JSON 형식으로, title / summary / tags 3개의 키만 포함해 주세요."

Output: JSON / Markdown / Table
Mistake 04

과도한 다중 요청

하나의 프롬프트에 10가지를 한꺼번에 요청하면 AI가 우선순위를 잃고 품질이 저하됩니다.

뒤집어서 해결책 보기
Solution

단계별 분리

작업을 3~4단계로 쪼개어 순서대로 요청하는 Chain-of-Thought 전략을 사용하세요.

Step 1 → Step 2 → Step 3
